# E课堂 IVD 大练兵数据处理模块

## 模块定位

本模块用于处理 IVD 春季、夏季、秋季、冬季大练兵考试数据，支持第一次考试、第一次补考、第二次补考或重修，输出成绩汇总 Excel，并可选生成 Word 分析报告。

- 前端模块：E课堂数据处理中的 `IVD + 大练兵`
- 后端 adapter：`backend/services/eclass/adapters/ivd_big_teach_adapter.py`
- 底层算法：`backend/legacy/eclass/big_teach.py`
- 公共调度：`backend/services/eclass/runner.py`

## 输入文件

上传槽位由 `schemas.py` 定义：

- `first_folder`：第一次考试文件夹，必填，支持 `.xlsx`、`.xls`、`.xlsm`
- `resit_folder`：第一次补考文件夹，可选
- `remake_folder`：第二次补考或重修文件夹，可选
- `template_doc`：Word 模板文件，可选，仅支持 `.docx`

支持三种业务场景：

- 只上传第一次考试。
- 上传第一次考试和第一次补考。
- 上传第一次考试、第一次补考、第二次补考或重修。

## adapter 流程

`ivd_big_teach_adapter.run` 负责将 E课堂上传框架与旧算法连接起来：

1. 从上传槽位中定位文件夹。
2. 统计第一次考试、第一次补考、第二次补考文件夹中的 Excel 数量。
3. 如果第一次考试没有 Excel，直接报错。
4. 从文件名或底层函数识别大练兵季节，默认兜底为 `春季`。
5. 根据季节生成输出文件名：
   - `IVD{季节}大练兵成绩汇总.xlsx`
   - `IVD用户服务工程师{季节}大练兵分析报告.docx`
6. 删除任务临时目录中同名旧文件。
7. 调用底层算法：
   - `merge_all_exam_data(first_folder, remake_folder, resit_folder)`
   - `generate_summary_excel(merged_data, excel_path)`
   - 如果上传了 Word 模板，再调用 `generate_analysis_report(excel_path, template_path, word_path)`
8. 收集算法 stdout 和 stderr 中的最近日志。
9. 返回输出文件、摘要、日志和 warning。

注意：底层函数参数顺序保持旧版兼容，是 `first_folder, remake_folder, resit_folder`。adapter 中已经按这个旧顺序传参。

## 底层数据读取逻辑

底层算法会读取三类考试文件夹：

- `first`：第一次考试。
- `resit`：第一次补考。
- `remake`：第二次补考或重修。

读取时会做这些处理：

- 自动识别文件中的表头行。
- 对字段做模糊重命名，尤其是账号、工号、姓名、部门、岗位、成绩。
- 如果没有工号但有账号，则用账号生成工号。
- 标准化工号，去掉空工号。
- 标记考试类型和考试科目。
- 过滤姓名中包含 `已禁用` 的人员。
- 从所有科目和考试类型中汇总人员基础信息，同一个人员保留更完整的姓名、部门和岗位。

## 科目合并逻辑

系统按标准科目顺序逐科处理。每个科目会：

1. 提取该科目所有考试记录。
2. 建立该科目出现过的所有人员列表。
3. 分别生成第一次考试、第一次补考、第二次补考成绩表。
4. 同一人员同一考试类型如果有多条成绩，保留最高分。
5. 对没有上传的考试类型显示为 `/`。
6. 对已经上传但某人没有成绩记录的考试类型按 0 分处理。
7. 根据工号判断员工类型：
   - 工号以 `S` 开头为渠道。
   - 其他为内部。
8. 从部门字段解析区域、分公司和渠道商。
9. 内部员工的渠道商显示为 `/`。
10. 岗位不是 `工程师` 或 `用户服务工程师` 时写入备注。
11. 如果找不到区域、分公司或渠道商，会写入异常人员清单。

## 分数计算逻辑

### 单科最终成绩

`calculate_final_score` 使用：

- 第一次考试成绩
- 第一次补考成绩
- 第二次补考成绩

规则：

- 如果没有任何补考或重修成绩，最终成绩取第一次考试成绩。
- 如果存在补考或重修成绩，取三次成绩最高值，但最高封顶 80 分。

这意味着补考可以帮助通过，但不会让最终成绩超过 80。

### 个人综合成绩

成绩汇总 sheet 中的 `平均成绩` 实际口径是个人综合成绩：

- 不再按科目设置权重。
- 不再计算算术平均。
- 取所有有效科目最终成绩中的最高分。
- 0、空值、`/` 不参与有效成绩判断。
- 如果没有任何有效正分科目，则个人综合成绩为 0。

### 缺考口径

员工所有已导入科目的最终成绩都为 0 或空时，视为缺考。

未导入的科目不影响缺考判断。只要任意一个已导入科目最终成绩大于 0，就不算缺考。

### 通过口径

区域、分公司和产品线分析中：

- 通过人数 = 个人综合成绩或单科最终成绩大于等于 80 的人数。
- 通过率 = 通过人数 / 总人数。
- 平均分不包含缺考人员。

## 输出工作簿

Excel 输出：`IVD{季节}大练兵成绩汇总.xlsx`

主要 sheet：

- 各考试科目 sheet：展示每科第一次、补考、重修和最终成绩。
- `成绩汇总`：按人员汇总区域、分公司、渠道商、工号、姓名、各科成绩、综合成绩、是否缺考和备注。
- `区域分析`：按区域统计总人数、平均分、通过人数、通过率、缺勤人数、缺勤率、内部和渠道通过情况。
- `分公司分析`：按分公司统计同类指标。
- `产品线分析`：按实际导入科目统计总人数、平均分、通过人数、通过率、内部和渠道通过情况，不展示缺勤指标。
- `异常人员` 或独立异常排查清单：用于排查区域、分公司、渠道商解析失败的人员。

如果上传了 Word 模板，还会生成：

- `IVD用户服务工程师{季节}大练兵分析报告.docx`

最终文件会被发布到：

- `Downloads/IVD大练兵数据统计结果`

## 业务价值

本模块把多次考试和多科目成绩统一成一套稳定的成绩汇总和管理分析口径，解决人工合并补考、重修、内部渠道拆分和缺考判断容易出错的问题。它适合用于大练兵活动复盘、区域和分公司能力对比、渠道学习效果跟踪以及分析报告生成。

## 维护注意点

- 补考封顶 80 是核心业务口径，不要误改成直接取最高分。
- `平均成绩` 字段名沿用旧模板，但实际是个人最高有效科目分，维护说明时要讲清楚。
- 科目展示取决于实际导入文件，产品线分析也只展示实际导入科目。
- 如果部门字段格式变化，优先维护底层的部门解析函数。
- 如果新增季节或改输出文件名，维护 `DRILL_SEASONS` 和 `get_drill_output_filenames`。
- 如果 Word 报告生成失败，先确认模板占位文本和 `python-docx` 依赖。
