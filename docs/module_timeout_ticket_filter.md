# 超时工单筛选模块

## 模块定位

本模块用于从工单报表中筛出仍处于待处理状态、且尚未进入质量上升报表前续工单记录的 IVD 工单。它是一个后端 Excel 计算模块，前端负责上传两个 Excel，后端负责清洗、比对、导出。

- 前端页面：`frontend/src/tools/TimeoutTicketTool.vue`
- 后端入口：`POST /api/process`
- 核心函数：`backend/main.py::process_excels`
- 下载入口：`GET /api/download/{job_id}`

## 输入文件

需要上传两个文件：

- 工单报表：系统会读取第一个 sheet，识别工单主数据。
- 质量上升报表：系统会读取第一个 sheet，识别已进入质量流程的前续工单。

字段采用别名匹配：

- 工单大产线：`大产线`、`产品线`
- 工单状态：`工单状态`、`状态`
- 工单号：`工单号`、`服务工单号`、`工单编号`
- 质量前续工单：`前续工单`、`工单号`、`服务工单号`、`前续工单号`

## 核心计算逻辑

1. 读取工单报表和质量上升报表。
2. 对表头进行别名识别，找到大产线、状态、工单号和前续工单字段。
3. 对工单报表做基础筛选：
   - 大产线必须等于 `IVD`。
   - 工单状态必须属于 `服务执行中`、`已派工`、`已预约`。
4. 对质量上升报表的前续工单字段做拆分和标准化。
   - 支持一个单元格中出现多个工单号。
   - 会按逗号、中文逗号、分号、空白符等分隔。
   - 会统一去空格、去无效值，形成前续工单号集合。
5. 对待处理 IVD 工单的工单号做标准化。
6. 将已经出现在质量上升报表前续工单集合中的工单排除。
7. 剩余数据即为“待关注且尚未进入质量上升前续链路”的工单。

## 输出结果

后端会生成一个 Excel：

- 文件名：`工单报表-已筛选-{job_id}.xlsx`
- 下载时默认名称：`工单报表-已筛选.xlsx`
- Sheet 名称：`工单报表 - 已筛选`

接口返回的统计信息包括：

- 工单报表总行数
- 质量上升报表总行数
- IVD 待处理工单数
- 已匹配质量前续工单数
- 最终筛选结果数
- 实际匹配到的字段名
- 前 10 行预览数据

## 业务价值

本模块帮助快速定位“还在执行、派工或预约状态，但没有进入质量上升前续记录”的 IVD 工单。它适合用于超时跟进、质量闭环检查和服务过程漏斗排查，减少人工在两张报表之间逐条复制工单号核对的工作量。

## 维护注意点

- 如果上游报表字段改名，优先维护 `WORK_ORDER_ALIASES` 和 `QUALITY_ALIASES`。
- 如果需要扩展产线，修改 `TARGET_PRODUCT_LINE` 或将其改造成前端可选参数。
- 如果需要调整待处理状态，修改 `TARGET_STATUSES`。
- 工单号比对依赖 `_normalize_ticket_id` 和 `_expand_ticket_ids`，若发现工单号格式变化，应优先检查这两个辅助函数。
- 当前逻辑只做集合排除，不会输出“被质量上升报表命中的明细”。如后续业务需要追溯原因，可增加一个匹配明细 sheet。
