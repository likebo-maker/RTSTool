# 中国区培训覆盖地图模块

## 模块定位

本模块是前端本地解析和聚合模块，用于导入面授课程培训数据，按分公司、大区、产线、培训周期、培训结果、培训中心和培训类型展示培训覆盖与效果。它不依赖后端计算接口。

- 前端页面：`frontend/src/tools/TrainingCoverageMap.vue`
- 解析逻辑：`frontend/src/utils/trainingParser.js`
- 聚合逻辑：`frontend/src/utils/trainingAggregator.js`
- 结果归一化：`frontend/src/utils/trainingStatusNormalizer.js`
- 导出逻辑：`frontend/src/utils/exportTrainingExcel.js`
- 本地缓存：`frontend/src/services/localDataStore.js`

## 输入文件

支持一次导入一个或多个 Excel 文件。系统会扫描每个工作簿的所有 sheet。

字段采用别名匹配：

- 分公司：`区域`、`分公司`、`所属分公司`、`服务区域`
- 大区：`大区`、`区域大区`、`所属大区`、`班次所在大区`
- 产线：`产线`、`产品线`、`小产线`、`业务线`
- 培训年度：`培训年度`、`年度`、`年份`
- 培训月份：`培训月份`、`月份`
- 培训结果：`完成情况`、`成绩结果`、`培训结果`、`是否合格`、`状态`
- 培训中心或地点：`培训中心`、`举办地点`、`培训地点`
- 培训类型：`培训类型`、`课程类型`、`培训形式`、`课程形式`
- 课程名称：`培训名称`、`课程名称`
- 讲师：`讲师`、`讲师账号/姓名/组织`
- 学员姓名：`学员姓名`、`姓名`
- 学员组织：`学员组织`、`学员组织名称`
- 成绩：`成绩`
- 课时：`课时`
- 培训开始时间：`培训开始时间`、`培训开始日期`
- 培训结束时间：`培训结束时间`、`培训结束日期`
- 班次 ID：`班次ID`
- 培训组织方：`培训组织方`

## 解析逻辑

1. 使用 `xlsx` 在浏览器读取 Excel 文件。
2. 每个 sheet 转换为二维矩阵。
3. 在前 12 行中寻找表头。
   - 表头会去空格、括号、斜杠、下划线等符号后做别名匹配。
   - 命中至少 4 类字段才认为是有效表头。
4. 建立字段到列索引的映射。
5. 必要字段检查：
   - 分公司
   - 产线
   - 培训结果
   - 培训年度或培训开始时间
   - 培训月份或培训开始时间
6. 逐行清洗：
   - 空行跳过。
   - 分公司做标准化，并映射大区。
   - 培训地点解析到培训中心和培训中心城市。
   - 培训结果归一化为 `合格`、`不合格`、`未知`。
   - 培训周期优先由年度和月份拼成 `YYYY-MM`，缺失时从培训开始时间解析。
   - 生成班次 key，优先使用 `班次ID`，否则用培训周期、地点、类型、课程名和产线组合。
7. 生成标准记录对象，包含学员、分公司、大区、产线、周期、培训中心、培训类型、课程、讲师、成绩、课时、来源文件和来源行。

## 培训结果归一化

`trainingStatusNormalizer.js` 中定义了结果识别：

- 合格关键词：`合格`、`通过`、`已通过`、`pass`
- 不合格关键词：`不合格`、`未通过`、`fail`、`未合格`
- 其他或空值：`未知`

注意：系统先判断不合格关键词，因为 `不合格` 中包含 `合格`。

只有 `合格` 和 `不合格` 进入有效结果样本；`未知` 不参与合格率分母。

## 聚合计算逻辑

### 筛选条件

支持按以下维度筛选：

- 分公司
- 大区
- 产品线
- 培训周期
- 培训结果
- 培训中心
- 培训类型

### 总览指标

筛选后计算：

- 培训人次：按 `学员姓名|分公司|学员组织` 去重；如果没有姓名，则退化为记录数。
- 记录数：筛选后的原始记录数。
- 培训场次：按班次 key 去重。
- 合格率：合格数 / 有效结果数，保留 1 位小数；没有有效结果时显示 `暂无`。
- 不合格数：不合格记录数。

### 分公司指标

按分公司分组：

- 培训人次。
- 记录数。
- 培训场次。
- 合格率。
- 不合格数。
- 主要产线：记录数最多的前 3 个产线。
- 主要培训类型：记录数最多的前 3 个培训类型。
- 是否有有效结果：用于判断风险着色。

### 培训中心指标

按培训中心分组，计算逻辑与分公司类似：

- 培训人次
- 记录数
- 培训场次
- 合格率
- 不合格数
- 主要产线
- 主要培训类型

培训中心为 `未知培训中心` 时不会生成地图点位。

### 排名和趋势

- 分公司覆盖 TOP10：按记录数降序，培训人次作为次排序。
- 风险分公司 TOP10：按不合格数降序，合格率低的优先。
- 产品线分布：按产线计数，取前 8。
- 培训类型分布：按培训类型计数，取前 8。
- 培训趋势：按培训周期统计记录数、培训人次和合格率。

### 地图点位和颜色

地图点位来自培训中心坐标映射。

地图颜色有多种模式：

- 培训人次模式：人次大于等于 120 为良好，大于等于 60 为关注，否则普通。
- 培训场次模式：场次大于等于 20 为良好，大于等于 8 为关注，否则普通。
- 风险模式：不合格数大于等于 12 为高风险，大于等于 4 为关注。
- 合格率模式：没有有效结果为普通，合格率大于等于 90% 为良好，大于等于 70% 为关注，低于 70% 为高风险。

## 输出和缓存

模块支持：

- 导出当前筛选结果：`中国区培训覆盖筛选结果.xlsx`
- 导出某个分公司或培训中心明细：`{名称}_培训明细.xlsx`
- 将导入数据保存到 IndexedDB。
- 如果 IndexedDB 不可用，降级保存到 localStorage。

## 业务价值

本模块让培训管理人员快速判断全国培训覆盖是否均衡、哪些产线和分公司参与不足、哪些培训中心承担更多课程，以及不合格风险集中在哪里。它适合用于培训复盘、资源调配、分公司能力建设和培训质量追踪。

## 维护注意点

- 如果上游字段改名，维护 `trainingParser.js` 中的 `HEADER_ALIASES`。
- 如果培训结果出现新的表达方式，维护 `trainingStatusNormalizer.js` 的合格和不合格关键词。
- 如果培训中心坐标或别名变化，维护 `trainingCenterMap.js`。
- 当前培训人次按姓名、分公司、学员组织去重。如果后续有员工号，应改为员工号优先。
- 当前解析和聚合都在浏览器完成，大文件性能取决于用户电脑和浏览器。
