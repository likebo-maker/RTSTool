# 在线服务考核指标模块

## 模块定位

本模块用于从 MSP、MCC、视频服务、CRM 视频和每日质检记录中计算在线服务考核指标。它按分公司、整体和分包商三个层级输出结果，并在工作簿中写入计算逻辑说明。

- 前端页面：`frontend/src/tools/OnlineAssessmentTool.vue`
- 后端入口：`POST /api/online-assessment/process`
- 核心函数：`backend/main.py::process_online_assessment_excels`
- 下载入口：`GET /api/online-assessment/download/{job_id}`

## 输入文件

需要上传六个文件：

- MSP 工单总表，优先读取 sheet：`MSP工单原始数据`
- MCC 通话记录
- 视频服务记录
- MCC 热线工单
- CRM 视频工单
- 每日质检记录表，优先读取 sheet：`质检表格`

关键字段：

- MSP：`工单状态`、`分公司`、`申告渠道`、`首次派单时间`、`预约完成时间`、`服务结束时间`
- MCC 通话记录：`相关客服`、`响铃时间`
- 视频服务记录：`客服工号`、`振铃时长`、`是否接通`、`失败原因`、`通话时间`、`评价`、`排队时长`
- MCC 热线工单：`受理客服`、`创建时间`、`关闭时间`、`短信满意度结果`、`受理单(service call)状态`
- CRM 视频工单：`负责GTS编号`、`负责GTS`、`用户状态`
- 每日质检记录表：`代理商`，以及字段名前缀为 `扣分` 的列

## 基础口径

在线服务渠道包括：

- `热线申告`
- `微信视频申告`
- `一键报修申告`
- `微信申告`
- `远程申告`

分包商固定统计：

- 迪诺
- 科海
- 庆余堂
- 尚肯

分包商识别有两套规则：

- 文本匹配：从相关客服、受理客服、代理商、负责 GTS 等字段中查找分包商名称、人员姓名或别名。
- 工号映射：从视频客服工号、负责 GTS 编号中识别员工号，再映射到分包商。

## 核心计算逻辑

### 1. 分公司指标

按 MSP 工单总表中的 `分公司` 分组：

- 工单总数量 = 当前分公司所有 MSP 工单数量。
- 已取消工单数量 = 当前分公司 `工单状态` 等于 `已取消` 的数量。
- 分公司工单取消率 = 已取消工单数量 / 工单总数量。

### 2. 整体指标

先筛选申告渠道属于在线服务渠道的 MSP 工单。

在线服务工单预约及时率：

- 分母 = 在线服务渠道且工单状态不是 `已取消` 的工单数量。
- 分子 = 预约完成时间 - 首次派单时间小于等于 30 分钟的工单数量。

平均修复时长（线上）：

- 分子 = 在线服务渠道工单中，服务结束时间 - 首次派单时间的总时长。
- 分母 = 在线服务渠道工单数量。
- 结果格式化为小时。

### 3. 分包商指标

#### 热线 15 秒接起率

- 数据源：MCC 通话记录。
- 分包商匹配字段：`相关客服`。
- 分子 = `响铃时间` 小于等于 15 秒的记录数。
- 分母 = 当前分包商 MCC 通话记录总数。

#### 视频 15 秒接起率

- 数据源：视频服务记录。
- 分包商匹配字段：`客服工号`。
- 分子 = `是否接通` 为 `是` 且 `振铃时长` 小于等于 15 秒。
- 分母由四类记录组成：
  - 接通且振铃时长小于等于 15 秒。
  - 接通且振铃时长大于 15 秒。
  - 未接通且振铃时长大于等于 30 秒。
  - 未接通且失败原因为 `canceled` 且振铃时长大于等于 15 秒。

#### 平均在线工单处理时长

- 热线时长 = MCC 热线工单的 `关闭时间 - 创建时间`。
- 视频时长 = 视频服务记录的 `通话时间`。
- 分母 = MCC 热线工单数 + 视频服务记录数。
- 结果 = 热线总时长 + 视频总时长 / 分母。

#### 满意度

热线满意度：

- 数据源：MCC 热线工单。
- 评分字段：`短信满意度结果`。
- 1 分档折算 100 分，2 分档折算 80 分，3 分档折算 60 分。
- 满意度 = 折算分总和 / 样本数 / 100。

视频满意度：

- 数据源：视频服务记录。
- 评分字段：`评价`。
- 只纳入 1 到 5 分的有效评价。
- 满意度 = 评价总分 / 有效样本数 / 5。

#### 质检合格率

- 数据源：每日质检记录表。
- 分包商匹配字段：`代理商`。
- 扣分字段为空视为合格。
- 质检合格率 = 合格数 / 质检总数。

#### 热线解决率

- 数据源：MCC 热线工单。
- 分包商匹配字段：`受理客服`。
- 分子 = `受理单(service call)状态` 等于 `技术支持解决` 的数量。
- 分母 = 状态属于 `技术支持解决` 或 `派工完成` 的数量。

#### 视频解决率

- 数据源：CRM 视频工单。
- 分包商匹配字段：`负责GTS编号`、`负责GTS`。
- 分子 = `用户状态` 等于 `技术支持解决` 的数量。
- 分母 = 状态属于 `技术支持解决` 或 `转FSM平台派工` 的数量。

#### 视频占线率

- 数据源：视频服务记录。
- 分子 = `排队时长` 大于 3 秒的视频记录数。
- 分母 = 当前分包商视频记录总数。

## 输出结果

输出文件：`在线服务考核指标计算表_{job_id}.xlsx`

工作簿包括：

- 源表 MSP 工单总表
- 源表 MCC 通话记录
- 源表 视频服务记录
- 源表 MCC 热线工单
- 源表 CRM 视频工单
- 源表 每日质检记录
- 计算结果，包含分公司指标、整体指标、分包商指标
- 计算逻辑，说明各指标来源和公式

接口返回预览默认展示分包商指标。

## 业务价值

本模块把服务时效、接通、满意度、质量、解决率和占线风险放在同一套考核口径下，适合用于分包商月度考核、在线服务运营复盘和服务质量问题定位。它能把原先需要多张报表人工汇总的指标自动化，减少口径不一致。

## 维护注意点

- 分包商别名和员工号映射在 `ASSESSMENT_PROVIDER_ALIASES` 和 `ASSESSMENT_STAFF_PROVIDER_MAP` 中维护。
- 如果新增分包商，需要同步增加分包商列表、别名和工号映射。
- 时长字段通过 `_duration_to_seconds` 统一解析，支持数值、秒、min、`HH:MM:SS` 和 `MM:SS`。
- 质检扣分列按字段名前缀查找，如果上游字段不再以 `扣分` 开头，需要修改查找逻辑。
- 视频 15 秒接起率的分母是业务定制口径，不等于视频服务记录总数，维护时不要误改成简单总量。
