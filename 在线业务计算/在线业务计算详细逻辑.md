# 在线业务计算详细逻辑

> 目的：本笔记用于把《在线业务计算》中的人工 Excel 口径整理为完整、清晰、可开发的规格说明。后续可直接将本笔记交给 AI 或开发人员，用于开发“在线业务计算”自动化程序。

---

## 1. 文档目标

### 1.1 要实现的目标

本笔记需要说清楚以下内容：

1. 在线业务计算需要读取哪些原始 Excel；
2. 每个 Excel 中哪些 Sheet 和字段参与计算；
3. 每个指标的分子、分母分别怎么筛选；
4. 哪些指标按工单数量统计，哪些指标按客户 ID 去重统计；
5. 当前 2026 年样例数据下的中间值和最终结果；
6. 程序开发时需要做哪些输入校验、异常处理和输出设计。

### 1.2 重要边界

1. 不修改原始 Excel 文件；
2. 不修改 Obsidian 原笔记《在线业务计算》；
3. 本文档只在统计目录下作为详细开发规格使用；
4. 本文档中的样例值基于当前原始数据文件计算得到；
5. 后续如果业务口径变化，应优先修改本文档中的“口径确认清单”和相关公式。

## 4. 通用处理规则

### 4.1 年份筛选

程序应支持传入目标年份，例如 `2026`。

| 数据源 | 年份字段 | 处理方式 |
|---|---|---|
| MCC热线原始数据 | `创建时间` | 转日期后取年份 |
| 视频工单原始数据 | `创建日期` | 转日期后取年份 |
| MSP工单原始数据 | `创建时间` | 转日期后取年份 |
| IVD客户群数量2025 | 不按工单年份筛选 | 直接使用客户群文件中的客户数量 |

规则：

1. 日期字段转换失败的数据，不计入年度工单类指标；
2. 日期为空的数据，不计入年度工单类指标；
3. 客户群文件当前不按日期筛选，默认文件即代表当前客户群口径。

### 4.2 字符串标准化

程序读取 Excel 后，应对字段名和文本值做标准化：

1. 字段名去除首尾空格；
2. 文本值去除首尾空格；
3. 空字符串转换为空值；
4. 状态和渠道使用精确匹配，不建议模糊匹配。

### 4.3 空值处理

| 场景 | 处理规则 |
|---|---|
| `客户ID` 为空 | 覆盖率分子去重时不计入客户数 |
| `VIP客户级别` 为空 | 视为 JC+社办候选客户 |
| 状态字段为空 | 不满足任何状态集合 |
| 渠道字段为空 | 不满足任何渠道集合 |
| `客户数量` 为空或非数字 | 求和时忽略或按 0，并写入异常检查 |
| 指标分母为 0 | 结果输出 `N/A`，备注写明分母为 0 |

## 6. 公共分母逻辑

“年度热线工单占比”和“年度热线整体解决率”使用相同公共分母。

```text
公共分母 = A + B + C
```

其中：

```text
A = 年度热线进线量 + 年度视频转技术支持工单数量
B = 年度 MSP 远程申告工单数量
C = 年度 MSP CS申告工单数量
```

### 6.1 A：年度热线进线量 + 年度视频转技术支持工单数量

#### 6.1.1 年度热线进线量

数据来源：`MCC热线原始数据.xlsx`

筛选条件：

```text
创建时间年份 = 目标年份
且
申告渠道 in ["001_热线申告", "006_微信申告", "009_一键报修申告"]
```

当前 2026 样例：

```text
年度热线进线量 = 4583 + 100 + 535 = 5218
```

#### 6.1.2 年度视频转技术支持工单数量

数据来源：`视频工单原始数据.xlsx`

筛选条件：

```text
创建日期年份 = 目标年份
且
用户状态 in ["技术支持解决", "无需技术支持", "转FSM平台派工", "座席解决"]
```

当前 2026 样例：

```text

```

#### 6.1.3 A 的计算

### 6.2 B：年度 MSP 远程申告工单数量

## 7. 指标一：年度热线工单占比

### 7.1 指标含义

年度热线工单占比用于衡量 MCC 热线转技术支持相关工单量，在整体在线业务工单量中的占比。

### 7.2 分子

数据来源：`MCC热线原始数据.xlsx`

筛选条件：

```text
创建时间年份 = 目标年份
且
受理单(service call)状态 in ["技术支持解决", "派工完成", "取消", "无需技术支持"]
```

计算方式：

```text
年度热线工单占比分子 = 满足条件的 MCC 工单行数
```

当前 2026 样例：

```text
年度热线工单占比分子 = 1812 + 2893 + 7 + 391 = 5103
```

### 7.3 分母

使用第 6 章定义的公共分母。

当前 2026 样例：

```text
年度热线工单占比分母 = 39030
```

### 7.4 公式

```text
年度热线工单占比 = 年度热线工单占比分子 / 公共分母
```

当前 2026 样例：

```text
年度热线工单占比 = 5103 / 39030 = 0.130746...
```

按百分比展示：

```text
年度热线工单占比 ≈ 13.07%
```

## 15. 推荐计算流程

### 15.1 读取文件

1. 读取 MCC 热线数据；
2. 读取视频工单数据；
3. 读取 MSP 工单数据；
4. 读取 IVD 客户群 3 个 Sheet。

### 15.2 标准化数据

1. 字段名去首尾空格；
2. 文本值去首尾空格；
3. 空字符串转为空值；
4. 日期字段转为日期；
5. `客户数量` 转为数字。

### 15.3 年度过滤

1. MCC 按 `创建时间` 过滤目标年份；
2. 视频按 `创建日期` 过滤目标年份；
3. MSP 按 `创建时间` 过滤目标年份；
4. IVD 客户群不做年度过滤。

### 15.4 计算中间量

依次计算：

1. MCC 热线进线量；
2. 视频转技术支持工单数量；
3. MSP 远程申告数量；
4. MSP CS申告数量；
5. 公共分母；
6. 热线技术支持已解决数量；
7. 覆盖客户去重数量；
8. IVD 客户群分母。

### 15.5 计算指标

依次计算：

1. 年度热线工单占比；
2. 年度热线整体解决率；
3. 热线解决率；
4. 整体客户覆盖率；
5. JC+社办覆盖率。

## 17. 异常检查规则

程序应至少输出以下异常检查结果：

| 异常类型 | 检查方式 | 建议处理 |
|---|---|---|
| 缺少文件 | 检查 4 个必需 Excel 是否存在 | 终止计算并提示 |
| 缺少 Sheet | 检查必需 Sheet 是否存在 | 终止计算并提示 |
| 缺少字段 | 检查关键字段是否存在 | 终止计算并提示 |
| 日期无法解析 | 日期转化为 `NaT` 的行 | 输出异常明细 |
| `客户ID` 为空 | 覆盖率候选数据中客户ID为空 | 不计入去重分子，输出异常明细 |
| `客户数量` 非数字 | IVD `客户数量` 转数字失败 | 不计入求和，输出异常明细 |
| 分母为 0 | 指标计算前判断 | 输出 `N/A`，备注分母为 0 |
| 筛选结果为 0 | 各关键中间量为 0 | 输出提示，供人工复核 |

## 18. 人工复核重点

人工复核时优先检查：

1. 热线渠道是否为 `001_热线申告`、`006_微信申告`、`009_一键报修申告` 三项；
2. 视频转技术支持数量是否包含 `座席解决`；
3. 年度热线工单占比分子状态是否不包含 `座席解决`；
4. 年度热线整体解决率和热线解决率的分子是否同时限定 `技术支持解决` 和热线渠道；
5. 公共分母是否等于 `热线进线量 + 视频转技术支持数量 + MSP远程申告数量 + MSP CS申告数量`；
6. 客户覆盖率分子是否按 `客户ID` 去重，而不是按工单行数统计；
7. JC+社办覆盖率分子是否筛选了 `VIP客户级别` 为空；
8. 覆盖率分子状态是否不包含 `座席解决`。

   - Sheet1：MCC热线原始数据
   - Sheet2：视频工单原始数据
   - Sheet3：MSP工单原始数据
   - Sheet4：IVD客户群数量2025
4. 第 5 个 Sheet 是 MCC热线数据按客户ID去重后的表格，Sheet 名建议为：MCC热线客户去重明细。
5. 第 5 个 Sheet 的生成规则：
   - 数据来源为 MCC热线原始数据.xlsx。
   - 先按文档中的覆盖率分子状态集合筛选 MCC 数据，即受理单(service call)状态 in [技术支持解决, 派工完成, 取消, 无需技术支持]。
   - 排除客户ID为空的数据。
   - 按客户ID去重，每个客户ID只保留一行。
   - 去重后，需要在“客户ID”列后面插入一列“客户层级”。
6. “客户层级”只有 3 个可选值：
   - 双大
   - 县域+社办AB类
   - JC+社办CD类
7. “客户层级”的判断规则：
   - 用第 5 个 Sheet 中的客户ID，去 IVD客户群数量2025.xlsx 中匹配客户代码。
   - 先匹配 IVD客户群数量2025.xlsx 的“双大”Sheet。
   - 如果客户ID 能匹配到“双大”Sheet 中的客户代码，则客户层级 = 双大。
   - 如果没有匹配到“双大”，再匹配“县域+社办AB类”Sheet。
   - 如果客户ID 能匹配到“县域+社办AB类”Sheet 中的客户代码，则客户层级 = 县域+社办AB类。
   - 如果“双大”和“县域+社办AB类”都匹配不到，则客户层级 = JC+社办CD类。
8. 第 6 个 Sheet 是最终计算结果表，Sheet 名建议为：计算结果。
9. “计算结果”Sheet 至少输出以下内容：
   - 指标名称
   - 分子说明
   - 分子值
   - 分母说明
   - 分母值
   - 计算公式
   - 小数结果
   - 百分比结果
   - 备注
10. 输出文件不得修改 4 个输入原始 Excel 文件。

六、校验和异常处理要求

1. 程序必须校验 4 个输入 Excel 是否存在。
2. 程序必须校验必需 Sheet 和必需字段是否存在。
3. 程序必须处理日期无法解析、客户ID为空、客户数量非数字、分母为0等异常。
4. 如果分母为 0，对应指标结果输出 N/A，并在备注中说明“分母为0，无法计算”。
5. 百分比默认保留 2 位小数。
6. 如果文件缺失、字段缺失或 Sheet 缺失，页面要在操作控制卡片中显示清晰错误提示。
