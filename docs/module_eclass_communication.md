# E课堂 IVD 交流会数据处理模块

## 模块定位

本模块用于处理 IVD 技术交流会六科考试成绩，自动识别文件、从成绩表反推应出勤名单、计算出勤和考核情况，生成主统计 Excel、附录 Excel 和 Word 总结邮件。

- 前端模块：E课堂数据处理中的 `IVD + 交流会`
- 后端 adapter：`backend/services/eclass/adapters/ivd_communication_adapter.py`
- 底层算法：`backend/legacy/eclass/Comunication_meeting.py`
- 公共调度：`backend/services/eclass/runner.py`

## 输入文件

上传槽位由 `schemas.py` 定义：

- `data_folder`：交流会数据文件夹，必填，支持 `.xlsx`、`.xls`
- `template_doc`：Word 模板文件，必填，仅支持 `.docx`

数据文件夹中需要能识别六科成绩文件：

- 凝血
- 化免
- 临检
- TLA
- 微生物
- 尿液

## 文件自动识别逻辑

`detect_communication_files` 只扫描交流会数据文件夹当前层级，不递归子目录。

识别规则：

- 忽略 `~$` 开头的 Office 临时文件。
- 只纳入 `.xlsx` 和 `.xls`。
- 文件名包含科目名即认为是候选文件。
- `TLA` 使用大小写不敏感匹配。
- 如果同一科目有多个候选文件，按以下顺序选择最新：
  - 文件名中的日期，例如 `2026-05-20`、`2026_05_20`、`2026.05.20`、`2026年5月20日`
  - 文件名中的 14 位时间戳
  - 文件修改时间
  - 文件名

如果六科任一文件未识别到，处理会中止并提示缺少的科目。

## adapter 流程

`ivd_communication_adapter.run` 的流程：

1. 定位上传的数据文件夹。
2. 校验 Word 模板是否上传。
3. 自动识别六科成绩文件。
4. 准备三个输出文件：
   - `交流会统计结果.xlsx`
   - `中国区技术交流会出勤和考核情况-IVD线.xlsx`
   - `中国区技术交流会总结邮件.docx`
5. 删除任务临时目录中的同名旧文件。
6. 调用底层 `communication.main(...)`。
7. 收集底层日志。
8. 返回三个输出文件和 overall_summary。

最终文件会被发布到：

- `Downloads/IVD交流会数据统计结果`

## 单科成绩读取逻辑

底层 `load_exam_scores` 会对每科成绩文件做标准化：

1. 自动识别表头行，要求至少包含 `账号`、`部门`、`成绩`。
2. 必需字段包括 `账号`、`姓名`、`部门`、`成绩`、`交卷时间`。
3. 账号标准化为工号。
4. 从部门解析分公司和渠道商。
5. 工号以 `S` 开头识别为渠道，其余为内部。
6. 读取原始成绩。
7. 从交卷时间中推断本次考试派发日期。
8. 计算单科最终成绩。
9. 根据交卷时间和考试状态判断是否有有效交卷记录。
10. 记录 `present_subjects`，表示该人员出现在该科原始成绩表中。
11. 同一个工号在同一科多条记录时，优先保留有效交卷记录，再按成绩高低保留。

## 单科最终成绩逻辑

底层逻辑会根据交卷时间和派发日期调整最终成绩：

- 派发当天考试：最终成绩等于原始成绩。
- 派发日期之后考试：如果原始成绩大于等于 80，最终成绩封顶为 80；如果低于 80，保留原始成绩。
- 未开始、未提交或无交卷时间：最终成绩为 0。

这个口径用于支持补做或延后考试只达到通过，不拉高高分。

## 名单生成逻辑

当前不再要求上传 `名单.xlsx`。名单由六科成绩表中的所有账号自动去重汇总。

`build_roster_from_exam_dfs` 会：

- 合并六科成绩中的人员基础信息。
- 按工号去重。
- 对同一个工号保留信息最完整的一条记录。
- 从部门补齐分公司和渠道商。
- 内部员工渠道商置空。

## 合并成绩逻辑

`merge_attendance_with_scores` 会：

1. 以名单为基础表。
2. 为六科初始化成绩列。
3. 逐科把最终成绩合并到人员行。
4. 合并 `attended_subjects` 和 `present_subjects`。
5. 保留分公司、渠道商、人员类型等基础信息。
6. 再通过 `enrich_person_info_from_exam_dfs` 从六科成绩表回填更完整的人员信息。

## 出勤和通过逻辑

### 出勤

当前统一出勤口径：

- 六科最终成绩全部为 0 或没有成绩，则为不出勤。
- 只要任意一科最终成绩大于 0，就算实际出勤。

该口径不再依赖交卷时间或 `attended_subjects`，避免已出现在成绩表但状态字段异常时导致出勤判断不一致。

### 通过

整体通过率：

- 通过人数 = 六科任一最终成绩大于等于 80 的人数。
- 实际参考人数 = 至少一科最终成绩大于 0 的人数。
- 整体通过率 = 通过人数 / 实际参考人数。

### 分公司平均成绩

分公司出勤率及平均成绩：

- 应出勤人数 = 该分公司纳入统计的总人数。
- 实际出勤人数 = 至少一科成绩大于 0 的人数。
- 出勤率 = 实际出勤人数 / 应出勤人数。
- 个人平均成绩按实际出现在成绩表中的科目计算，0 分参与平均，`/` 不参与平均。
- 分公司平均成绩只统计实际出勤人员。

Word 中内部和渠道考核成绩图直接读取附录 Excel 中的分公司平均成绩，不再重新计算第二套口径。

## 未出勤名单逻辑

`generate_absentee_lists` 会：

- 按统一出勤口径找出不出勤人员。
- 内部未出勤输出：工号、姓名、分公司。
- 渠道未出勤输出：工号、姓名、分公司、渠道商。
- 同时返回中文 key 和英文 key，兼容旧代码。

## 输出文件

### 1. 主统计 Excel

文件名：`交流会统计结果.xlsx`

主要内容：

- 名单 sheet。
- 六科原始成绩 sheet，并在原始成绩列后插入最终成绩列。
- 内部成绩。
- 渠道成绩。
- 交流会成绩。
- 分公司出勤率及平均成绩。
- 未出勤名单相关 sheet。

### 2. 附录 Excel

文件名：`中国区技术交流会出勤和考核情况-IVD线.xlsx`

用于承载 Word 图表和附录明细口径，尤其是分公司出勤率和平均成绩。

### 3. Word 总结邮件

文件名：`中国区技术交流会总结邮件.docx`

Word 生成逻辑会：

- 替换总结文本中的出勤描述。
- 插入内部出勤率、渠道出勤率、内部成绩、渠道成绩图。
- 图表数据优先来自附录 Excel，保证和附录口径一致。

## 业务价值

本模块将交流会成绩文件从“六科散表”整理成可汇报、可追责、可归档的结果包。它解决了名单维护、人员去重、内部和渠道拆分、出勤判断、补做封顶和 Word 图表口径一致性问题，适合用于月度技术交流会复盘和区域培训效果跟踪。

## 维护注意点

- 当前只扫描数据文件夹当前层级，如果用户把成绩文件放在子目录，需要改为递归扫描。
- 文件自动识别依赖文件名包含科目名，命名不规范会导致识别失败。
- 出勤口径是六科任一最终成绩大于 0，不是交卷时间。
- 单科延后考试封顶 80 是核心业务规则，不要误改。
- Word 图表口径依赖附录 Excel 的分公司统计 sheet，修改附录列名时要同步修改 Word 读取逻辑。
- 如果新增科目，需要同步修改 adapter 的 `SUBJECTS`、底层 `exam_types` 和所有输出表结构。
