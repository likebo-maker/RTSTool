# 中国区人员服务资质地图模块

## 模块定位

本模块是前端本地解析和聚合模块，用于导入人员服务资质 Excel，在浏览器中计算资质有效状态、分公司风险、产品线和资质类型分布，并通过地图和图表展示。它不依赖后端计算接口。

- 前端页面：`frontend/src/tools/ServiceQualificationMap.vue`
- 解析逻辑：`frontend/src/utils/qualificationParser.js`
- 聚合逻辑：`frontend/src/utils/qualificationAggregator.js`
- 状态口径：`frontend/src/utils/qualificationTypes.js`
- 导出逻辑：`frontend/src/utils/qualificationExport.js`
- 本地缓存：`frontend/src/services/localDataStore.js`

## 输入文件

支持一次导入一个或多个 Excel 文件。系统会扫描每个工作簿的所有 sheet。

字段采用别名匹配：

- 分公司：`所属分公司`、`分公司`、`所属公司`
- 区域：`区域`、`大区`
- 产品线：优先 `产品线描述`，备用 `产品线`、`大产线`、`产线`、`部门`
- 机器型号：`机器型号`、`型号`、`机型`
- 服务资质类型：`服务资质类别描述`、`服务资质类型`、`资质类型`、`服务资质类别`
- 服务资质编码：`服务资质类别编码`
- 资质有效期：`资质有效期`、`有效期`、`截止日期`、`有效截止日期`、`资质有效截止日期`、`有效截止时间`
- 人员姓名：`人员姓名`、`姓名`、`员工姓名`、`工程师姓名`、`员工/分包商/经销商名称`
- 单位：`经销商`、`单位`、`分包商名称`、`员工/分包商/经销商名称`

系统会排除分公司名称为 `法国`、`荷兰`、`英国`、`总部` 的记录。

## 解析逻辑

1. 使用 `xlsx` 在浏览器读取 Excel 文件。
2. 每个 sheet 先转换成二维矩阵。
3. 在前 12 行中寻找最像表头的行。
   - 将表头去空格、括号、斜杠、下划线等符号后比较。
   - 命中至少 3 类字段才认为是有效表头。
4. 根据字段别名建立列索引。
5. 如果核心字段缺失过多，则跳过该 sheet 并记录 warning。
6. 逐行清洗数据：
   - 空行跳过。
   - 产品线优先取 `产品线描述`，如果是纯数字或空值，再取备用产品线字段。
   - 资质类型优先取描述字段，缺失时取编码字段。
   - 人员姓名缺失时用单位字段兜底，再缺失时写 `未命名人员`。
   - 分公司映射到大区，使用 `branchGeoMap` 的分公司和区域映射。
7. 调用 `buildQualificationStatus` 计算资质状态。
8. 生成标准记录对象，包含人员、分公司、产品线、型号、资质类型、有效期、状态、来源文件、来源 sheet 和来源行。

## 资质状态计算

资质有效期支持多种格式：

- `YYYY-MM-DD`
- `YYYY/MM/DD`
- `YYYY.M.D`
- `YYYY-MM-DD HH:mm:ss`
- `YYYY年M月D日`
- `M/D/YYYY`
- Excel 日期序列号
- JavaScript Date

以浏览器当前日期为参考日：

- 有效期早于今天：`已过期`
- 距离到期小于等于 30 天：`30天内到期`
- 小于等于 60 天：`60天内到期`
- 小于等于 90 天：`90天内到期`
- 大于 90 天：`有效`
- 日期无法识别：`未识别`

`有效`、`30天内到期`、`60天内到期`、`90天内到期` 都算当前有效资质；`已过期` 和 `未识别` 不算当前有效资质。

## 聚合计算逻辑

### 筛选条件

支持按以下维度筛选：

- 大区
- 分公司
- 产品线
- 机器型号
- 资质类型
- 资质状态

### 总览指标

筛选后计算：

- 人员总数：按 `人员姓名|分公司|单位` 去重。
- 有效资质总数：当前有效资质记录数。
- 即将到期资质：状态为 `30天内到期` 的数量。
- 已过期资质：状态为 `已过期` 的数量。
- 覆盖分公司数：筛选后有记录的分公司数量。

### 分公司指标

按分公司分组：

- 人员数：按人员去重。
- 有效资质数。
- 30 天、60 天、90 天内到期数。
- 已过期资质数。
- 主产品线：当前有效资质中数量最多的前 3 个产品线。
- 风险等级：
  - 高风险：存在已过期资质，或 30 天内到期资质数大于等于 `max(3, 总记录数 * 8%)`。
  - 关注：存在 30 天或 60 天内到期资质。
  - 正常：不满足以上条件。

### 图表数据

- 分公司有效资质 TOP10：按有效资质数降序，人员数作为次排序。
- 资质风险 TOP10：按已过期、30 天、60 天风险数量降序。
- 产品线资质分布：当前有效资质按产品线计数，取前 10。
- 资质类型分布：当前有效资质按资质类型计数。
- 到期趋势：30 天、60 天、90 天、已过期数量。

### 地图点位

分公司统计会通过 `resolveBranchGeo` 匹配城市和经纬度。未匹配到经纬度的分公司不会显示在地图上，但仍保留在表格和统计中。

## 输出和缓存

模块支持：

- 导出当前筛选结果：`中国区人员服务资质筛选结果.xlsx`
- 导出某个分公司资质详情：`{分公司}_资质详情.xlsx`
- 将导入后的记录和 warning 保存到 IndexedDB。
- 如果 IndexedDB 不可用，降级保存到 localStorage。

## 业务价值

本模块让维护者和业务方快速看到人员资质覆盖、资质风险和分公司薄弱点。它特别适合用于资质到期预警、分公司能力盘点、产品线服务覆盖检查和分包商资质治理。

## 维护注意点

- 如果上游字段改名，维护 `HEADER_ALIASES`。
- 如果需要调整资质到期阈值，维护 `qualificationTypes.js` 中的 `buildQualificationStatus`。
- 如果新增或修改分公司地图坐标，维护 `branchGeoMap.js`。
- 当前人员去重键是姓名、分公司、单位组合。如果将来有员工号字段，建议改为员工号优先。
- 当前所有计算都在浏览器中进行，大文件导入会受客户端性能影响。
