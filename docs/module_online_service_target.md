# 在线服务项目目标计算模块

## 模块定位

本模块用于汇总 MCC 热线、视频工单、MSP 工单和 IVD 客户群数据，计算年度在线服务解决率、工单占比、OEE 指标和客户覆盖率。它是一个后端 Excel 计算模块，输出带源表、计算结果、计算逻辑和仪表盘的工作簿。

- 前端页面：`frontend/src/tools/OnlineBusinessTool.vue`
- 后端入口：`POST /api/online-business/process`
- 核心函数：`backend/main.py::process_online_business_excels`
- 下载入口：`GET /api/online-business/download/{job_id}`

## 输入文件

需要上传四类文件：

- MCC 热线原始数据，要求 sheet：`MCC热线原始数据`
- 视频工单原始数据，要求 sheet：`视频工单原始数据`
- MSP 工单原始数据，要求 sheet：`MSP工单原始数据`
- IVD 客户群表，要求 sheet：`双大`、`县域+社办AB类`、`JC+社办CD类`

关键字段：

- MCC：`客户ID`、`客户名称`、`受理单(service call)状态`、`创建时间`、`申告渠道`、`VIP客户级别`
- 视频：`创建日期`、`用户状态`、`客户代码`、`客户`、`产品线`、`VIP客户级别`
- MSP：`创建时间`、`申告渠道`、`工作方式`、`工单类型`
- IVD 双大和 AB：`客户代码`、`客户名称`
- IVD JC：`客户数量`

## 基础口径

系统会自动从 MCC、视频、MSP 的时间字段中识别年份，并按年份逐年计算。

主要常量口径：

- MCC 热线渠道：`001_热线申告`、`006_微信申告`、`009_一键报修申告`
- MCC 覆盖状态：`技术支持解决`、`派工完成`、`取消`、`无需技术支持`
- 视频纳入状态：`技术支持解决`、`无需技术支持`、`转FSM平台派工`、`座席解决`
- 项目汇报解决状态：`无需技术支持`、`座席解决`、`坐席解决`、`技术支持解决`
- IVD MCC 大产线：`IVD`
- IVD 视频产品线：`血球`、`生化`
- MSP OEE 渠道：`远程申告`
- MSP OEE 已解决工作方式：`电话指导`、`远程解决`
- CS 申告渠道：`CS申告`
- CS 维修工单类型：`维修服务`
- CS 问题处理工单类型：`临床服务`

## 核心计算逻辑

### 1. 年度基础指标

每个年份会分别统计：

- 热线工单数量：该年份 MCC 数据总行数。
- 热线已解决工单数量：MCC 状态等于 `技术支持解决` 的数量。
- 热线项目解决分子：MCC 状态属于项目汇报解决状态的数量。
- 热线项目解决分母：项目汇报解决状态加 `派工完成` 的数量。
- 视频工单数量：该年份视频数据总行数。
- 视频已解决工单数量：视频状态等于 `技术支持解决` 的数量。
- 视频项目解决分子：视频状态属于项目汇报解决状态的数量。
- 视频项目解决分母：项目汇报解决状态加 `转FSM平台派工` 的数量。
- MSP OEE 工单数量：MSP 申告渠道为 `远程申告` 的数量。
- MSP OEE 已解决数量：MSP 申告渠道为 `远程申告` 且工作方式为 `电话指导` 或 `远程解决` 的数量。
- CS 维修数量：MSP 申告渠道为 `CS申告` 且工单类型为 `维修服务` 的数量。
- CS 问题处理数量：MSP 申告渠道为 `CS申告` 且工单类型为 `临床服务` 的数量。

### 2. 客户总量

客户总量来自 IVD 客户群表：

- 双大客户数量：`双大` sheet 行数。
- 县域加社办 AB 客户数量：`县域+社办AB类` sheet 行数。
- JC 加社办 CD 客户数量：`JC+社办CD类` sheet 中 `客户数量` 求和。
- IVD 总客户数量：双大客户数 + AB 客户数 + JC 客户数。

### 3. 热线覆盖

热线覆盖先筛 MCC：

- 年份匹配。
- 大产线等于 `IVD`。
- 状态属于 `技术支持解决`、`派工完成`、`取消`、`无需技术支持`。

然后按 `客户ID` 标准化去重，计算：

- 热线覆盖客户数量。
- 命中双大客户代码集合的数量。
- 命中县域加社办 AB 客户代码集合的数量。
- JC 覆盖数量 = 热线覆盖客户数量 - 双大覆盖数量 - AB 覆盖数量，最小为 0。
- 各类覆盖率 = 覆盖数量 / 对应客户总量。

### 4. 视频覆盖

视频覆盖先筛：

- 年份匹配。
- 产品线属于 `血球` 或 `生化`。

然后按 `客户代码` 标准化去重，计算整体、双大、AB、JC 覆盖数量和覆盖率。

### 5. 热线加视频联合覆盖

联合覆盖使用热线客户 ID 与视频客户代码的并集。

- 热线部分取 MCC IVD 覆盖客户 ID。
- 视频部分取 IVD 视频客户代码。
- 联合客户数量按去重后的并集计算。
- 双大、AB、JC 按客户代码集合命中情况拆分。

### 6. 解决率和占比

在线总量 = 热线工单数量 + 视频工单数量 + MSP OEE 数量 + CS 维修和问题处理数量。

主要指标：

- 热线工单占比 = 热线工单数量 / 在线总量。
- 视频工单占比 = 视频工单数量 / 在线总量。
- OEE 工单占比 = MSP OEE 数量 / 在线总量。
- 热线整体解决率 = 热线已解决工单数量 / 在线总量。
- 视频整体解决率 = 视频已解决工单数量 / 在线总量。
- MSP OEE 单整体解决率 = MSP OEE 已解决数量 / 在线总量。
- MSP OEE 工单解决率 = MSP OEE 已解决数量 / MSP OEE 工单数量。
- 在线解决率整体 = 热线已解决 + 视频已解决 + MSP OEE 已解决 / 热线工单 + 视频工单 + MSP OEE 工单。
- 热线解决率和视频解决率使用项目汇报解决分子和分母。

## 输出结果

输出文件：`在线服务项目目标计算表_{job_id}.xlsx`

工作簿包括：

- 源表 MCC、视频、MSP、IVD 客户表。
- MCC 热线客户去重明细。
- 计算结果，分为 `热线数据`、`视频数据`、`热线+视频数据` 三段。
- 计算逻辑，解释关键指标来源和公式。
- 仪表盘区域和图表，用于展示最新年份解决率、覆盖率和趋势。

接口返回的统计信息包括最新年份、客户总数、热线和视频覆盖数量、联合覆盖数量、文件行数等。

## 业务价值

本模块把分散在热线、视频、MSP 和客户群中的数据统一到项目目标口径中，帮助管理者判断在线服务转化能力、客户覆盖程度、不同服务方式解决效果和 OEE 贡献。它适合用于年度项目复盘、目标达成追踪和管理汇报。

## 维护注意点

- 如果产品线或状态口径调整，优先维护 `backend/main.py` 顶部的常量集合。
- 年份识别来自数据中的时间字段，若上游日期列改名，需要同步维护必需字段和年份检测逻辑。
- 视频覆盖当前按 `客户代码` 去重，热线覆盖按 `客户ID` 去重。若后续要跨系统客户统一，应新增统一客户映射表。
- JC 覆盖数量当前采用差额法，不直接按客户清单匹配。若 JC 客户表后续提供客户代码，应改为集合命中。
- 工作簿内有公式和样式，修改输出列顺序时要同步检查公式区域和仪表盘引用区域。
